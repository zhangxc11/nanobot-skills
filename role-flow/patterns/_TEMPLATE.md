# Pattern Template

> 所有 pattern 遵循此统一结构。新建 pattern 时复制此模板并填写各节。

---

## 适用场景

描述此 pattern 适合什么类型的任务/场景。包括：
- 任务复杂度（简单/中等/复杂）
- 任务类型（开发/评审/维护/...）
- 前置条件（需要什么才能启动此 pattern）

## 角色组合

列出参与的角色（引用 `roles/*/ROLE.md`）：
- 角色 A — 一句话说明在此 pattern 中的作用
- 角色 B — ...

## 流转规则

描述角色间的流转条件：

```
角色 A --[pass]--> 角色 B
角色 A --[fail]--> 处理方式（打回/阻塞/升级）
角色 B --[pass]--> 角色 C
...
```

每条规则说明：触发条件（verdict）、目标角色、附加条件（如有）。

## Cross-Check 覆盖矩阵

自证满足框架约束 1（每个产出都有另一角色 cross-check）：

| 产出角色 | 产出内容 | Cross-Check 角色 | 检查内容 |
|---------|---------|-----------------|---------|
| 角色 A | ... | 角色 B | ... |
| 角色 B | ... | 角色 C | ... |

> 此矩阵无遗漏即合规。如有豁免，必须标注豁免条件和理由。

## 验收标准

定义此 pattern 整体完成的条件：
- 所有角色 pass 的条件
- 最终交付物清单
- 质量门槛

## ⚠️ ATTENTION: Auditor 要求

此流程完成后**必须执行 Auditor**进行独立审计。跳过 Auditor 的风险由调用方承担。
