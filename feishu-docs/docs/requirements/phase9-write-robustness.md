## Summary

修复飞书文档写入工具的 5 个已知问题，全面提升文档写入可靠性。来源：上传 884 行设计文档时遇到多个问题，28 轮迭代才完成。

## 背景

上传大型 Markdown 文档到飞书时，遇到以下问题：
1. 嵌套列表导致 schema mismatch
2. 锚点链接导致写入失败
3. 大文档写入超时
4. 超时续传导致内容重复
5. 整行加粗标题格式渲染异常

## 方案（F9.1 ~ F9.5）

### F9.1 嵌套列表 schema mismatch 修复
- **现象**: 目录(TOC)中的嵌套列表（缩进子项）导致 chunk 报 `[1770006] schema mismatch`
- **根因**: 飞书 API 不支持嵌套列表的 `children` 字段
- **修复**: `md_to_blocks.py` 转换器自动扁平化嵌套列表

### F9.2 文档内锚点链接降级
- **现象**: `[text](#anchor)` 格式的锚点链接导致写入失败
- **修复**: 转换器自动将锚点链接降级为纯文本

### F9.3 大文档写入超时优化
- **现象**: 884 行文档写入过程中 600s 超时
- **修复**: 优化 chunk 大小或并发策略；skill 文档标注大文档推荐写入方式

### F9.4 续传内容重复修复
- **现象**: 超时后续传导致文档开头出现重复内容
- **修复**: `--resume-from` 精确定位续传起点，避免重复

### F9.5 加粗标题格式异常修复
- **现象**: 整行加粗标题（如 **方案 2: 后台执行**）格式渲染异常
- **来源**: cron-partition-redesign.md v5 上传时用户批注
- **修复**: 修复 Markdown 转飞书 block 时整行加粗文本的转换逻辑

## Glossary

| 术语 | 定义 |
|------|------|
| chunk | 飞书文档分段写入的单元 |
| schema mismatch | 飞书 API 拒绝不符合 block schema 的写入请求，错误码 1770006 |
| 续传 | 大文档超时后从断点继续写入 |
| md_to_blocks | feishu_doc.py 中将 Markdown 转换为飞书 document block 的模块 |
